# 规范: 测试文件分组规则 (Group Test Files Rule)

## ADDED Requirements

### 需求:测试文件分组规则 (Group Test Files Rule)

系统必须实现 `group-test-files` 审查规则，用于规范测试文件的存放位置。

#### 场景:存在测试目录时强制移动

- **当** 目录下存在 `test` 或 `__tests__` 子目录
- **且** 目录下存在 `*.test.ts` 或 `*.test.tsx` 文件（无论数量）
- **那么** 系统应建议将这些测试文件移动到 `test`（或 `__tests__`）目录中

#### 场景:不存在测试目录且测试文件较多时建议移动

- **当** 目录下不存在 `test` 和 `__tests__` 子目录
- **且** 目录下存在 2 个或更多 `*.test.ts` 或 `*.test.tsx` 文件
- **那么** 系统应建议创建一个 `test` 目录并将这些文件移动进去

#### 场景:不存在测试目录且测试文件较少时忽略

- **当** 目录下不存在 `test` 和 `__tests__` 子目录
- **且** 目录下只有 1 个 `*.test.ts` 或 `*.test.tsx` 文件
- **那么** 系统不应提出建议
