## 新增需求

### 需求:定义文件名命名规则配置 (FilenameNamingRule Config)

系统必须支持通过配置定义文件名命名规则。配置必须支持多组规则，每组规则针对特定的目录路径生效。
每组规则必须支持为不同的文件扩展名指定命名风格。
支持的命名风格必须包括：

- `camelCase`: 首字母小写的驼峰命名 (e.g., `myFile.ts`)
- `PascalCase`: 首字母大写的帕斯卡命名 (e.g., `MyFile.ts`)
- `snake_case`: 下划线连接命名 (e.g., `my_file.ts`)
- `kebab-case`: 横线连接命名 (e.g., `my-file.ts`)
- `regex:<pattern>`: 自定义正则表达式匹配

此外，必须支持配置文件名是否需要与所在目录名称一致（忽略扩展名）。

#### 场景:配置多组规则

- **当** 配置文件中定义了多组针对不同目录的规则
- **那么** 系统应根据文件所在目录应用最匹配的规则

#### 场景:检查文件命名风格

- **当** 文件名不符合配置的命名风格（如要求 `kebab-case` 但使用了 `PascalCase`）
- **那么** 系统应报告一个 `styling` 类型的问题

#### 场景:检查文件名与目录名一致性

- **当** 规则配置了 `matchDirectory: true` 且文件名（忽略扩展名）与目录名不一致
- **那么** 系统应报告一个 `structure` 类型的问题

### 需求:文件名命名规则检查 (FilenameNamingRule Check)

系统必须实现 `filename-naming-rule` 审查规则。该规则必须根据当前的配置对文件进行检查。
如果未找到匹配的规则配置，则跳过检查。

#### 场景:规则执行

- **当** 运行 `filename-naming-rule` 检查
- **那么** 它应遍历上下文中的所有文件，并根据配置验证文件名
